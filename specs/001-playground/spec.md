# Playground 页 - Spec v1.0

## 1. 概述

### 1.1 背景
开发者在日常工作中同时使用多个 AI 模型（OpenAI、Anthropic、DeepSeek 等），需要一个统一入口来对比和测试不同模型对同一 prompt 的响应。

### 1.2 目标
构建一个 Playground 页面，支持：
- 选择不同 AI 模型
- 输入 prompt 并发送
- 接收**结构化、流式渲染**的 AI 响应（思考过程 + 工具调用 + 最终答案）
- 查看原始结构化数据

### 1.3 范围
**本期包含**：
- 模型选择器（≥ 5 家）
- 多行输入 + 发送/清空
- 结构化内容的流式渲染（timeline 卡 + 表格卡 + markdown 卡）
- 骨架屏 loading 态
- 复制原始 JSON
- Edge Runtime API Route
- Vercel AI SDK v5 streamObject
- Zod schema 共享

**本期不包含**（v2+）：
- 历史对话 / 会话管理
- 对话分享链接
- Markdown 数学公式渲染（KaTeX）
- 跨刷新持久化（页面刷新后状态重置，prompt 和响应均不保留）

---

## 2. 用户故事

> 作为一个同时使用多个 AI 模型的开发者，我希望有一个 Playground 页面，
> 可以输入 prompt → 选择模型 → 看 AI 响应，
> 但 AI 响应不是单一文本块，而是**流式生成的结构化 UI 卡片**
>（比如思考过程卡 + 工具调用卡 + 最终答案卡）。

## 3. 用户场景与测试

### 场景 1：选择模型 → 输入 Prompt → 查看流式响应

1. 用户进入 Playground 页面
2. 在顶部模型选择器中选择目标模型（如 Claude 4.7）
3. 在多行输入框中键入 prompt
4. 按下 Cmd/Ctrl + Enter 或点击"发送"
5. 响应区**逐步**出现卡片：
   - 先出现"思考过程"时间线卡（骨架屏 → 填充）
   - 再出现"工具调用"表格卡（骨架屏 → 填充）
   - 最后出现"最终答案"markdown 卡（骨架屏 → 填充）
6. 每张卡片**独立**完成渲染，不等待其他卡片

**验证点**：
- 首个卡片在 2s 内开始渲染
- 三张卡片按逻辑顺序逐步出现
- 每张卡片填充过程有骨架屏过渡

### 场景 2：发送前清空

1. 用户在输入框中输入内容
2. 点击"清空"按钮
3. 输入框内容被清除，响应区重置

### 场景 3：复制原始 JSON

1. AI 响应完成后，用户点击"复制原始 JSON"按钮
2. 结构化 JSON 数据被复制到剪贴板
3. 用户可见反馈（如 toast 提示"已复制"）

### 场景 4：API 错误处理

1. 用户发送请求
2. 后端调用 AI API 时发生错误，UI 根据错误类型展示对应信息：
   - **鉴权错误（401/403）**：提示"API Key 无效或已过期"，引导用户检查环境变量配置
   - **速率限制（429）**：提示"请求过于频繁，请等待后重试"，展示 API 返回的 Retry-After 秒数
   - **上下文超长（400/413）**：提示"输入超出模型上下文限制，请缩短 prompt"
   - **网络/超时/其他**：提示"连接失败，请检查网络后重试"
3. 错误区域使用醒目颜色展示，不静默失败
4. 用户可修改输入或重试发送

**验证点**：
- 三类错误展示不同文案
- 速率限制展示剩余等待时间
- 错误状态下发送按钮恢复可用

### 场景 5：请求中防止重复提交

1. 用户点击"发送"后，发送按钮立即变为 disabled + loading 态
2. 模型选择器同时被禁用，防止切换
3. 用户无法再次点击发送，也无法切换模型
4. 响应完成（成功或失败）后，按钮和选择器恢复可用

### 场景 6：输入超长提示

1. 用户在输入框中输入内容
2. 输入框下方实时显示"已输入 N 字符 / 最大 M 字符"
3. 输入达到最大限制时，"发送"按钮被禁用，提示"超出上下文限制"
4. 用户删除部分文本后，按钮恢复可用

### 场景 7：响应式体验

1. 用户在 1280px+ 桌面屏上操作 → 完整布局
2. 用户缩放到 768px 平板屏 → 布局自适应

## 4. 功能需求

### FR-1: 模型选择器

| 字段 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 页面顶部展示模型选择器，至少包含 5 个选项 |
| 验收标准 | 下拉菜单中可见：GPT-5.3、Claude 4.7、Gemini 3.1 Flash、DeepSeek V3.2、Qwen 3.5 Flash；**请求进行中，选择器 disabled，响应完成后恢复** |

### FR-2: Prompt 输入区

| 字段 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 多行文本输入框 + 发送按钮 + 清空按钮 + 字符计数器 |
| 验收标准 | 支持多行输入；Cmd/Ctrl+Enter 触发发送；点击"清空"清除输入和响应区；输入框下方实时显示"已输入 N 字符 / 最大 M 字符"；**达到字符上限时发送按钮禁用** |

### FR-3: 结构化响应渲染

| 字段 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 响应区按流式顺序渲染三张独立卡片 |
| 验收标准 | 思考过程（timeline 卡）→ 工具调用（表格卡）→ 最终答案（markdown 卡），每张卡片逐步从骨架屏过渡到内容 |

### FR-4: 流式部分渲染

| 字段 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 每张卡片内的字段逐步到位，未就绪字段显示骨架屏 |
| 验收标准 | 卡片不等待内部所有字段到齐才渲染；每到达一个字段即更新对应区域 |

### FR-5: 复制原始 JSON

| 字段 | 描述 |
|------|------|
| 优先级 | P1 |
| 描述 | 提供按钮复制响应背后的结构化 JSON |
| 验收标准 | 点击按钮后 JSON 写入剪贴板，用户获得视觉反馈 |

### FR-6: 分类错误展示

| 字段 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | API 调用错误根据类型分类展示，不静默失败 |
| 验收标准 | 鉴权错误提示"API Key 无效或已过期"；速率限制提示"请求过于频繁"并展示剩余秒数；上下文超长提示"输入超出模型上下文限制"；网络/超时提示"连接失败"；错误状态下发送按钮恢复可用 |

### FR-7: 请求中状态管理

| 字段 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 请求进行中，发送按钮 disabled + loading 态，模型选择器 disabled |
| 验收标准 | 点击发送后按钮立即变 loading 态；模型选择器不可切换；响应完成（成功或失败）后两者恢复可用 |

### FR-8: 输入字符限制与计数

| 字段 | 描述 |
|------|------|
| 优先级 | P1 |
| 描述 | 输入框下方实时字符计数，超限时禁用发送按钮 |
| 验收标准 | 实时显示"已输入 N 字符 / 最大 M 字符"；达到上限时发送按钮 disabled 并提示"超出上下文限制"；删除后自动恢复 |

## 5. 非功能需求

### NFR-1: 性能

| 字段 | 描述 |
|------|------|
| 指标 | 首个 token 出现时间 < 2 秒 |
| 验证方式 | 从点击发送到首个卡片开始渲染的时间差 |

### NFR-2: 运行时

| 字段 | 描述 |
|------|------|
| 指标 | API Route 必须使用 Edge Runtime |
| 验证方式 | API Route 文件声明 `export const runtime = 'edge'` |
| 兼容性说明 | Edge Runtime 下仅支持支持标准 HTTP fetch 的 provider；OpenAI、Anthropic、DeepSeek 均兼容；部分 provider 可能需要通过自定义 base URL 适配 |

### NFR-3: 流式协议

| 字段 | 描述 |
|------|------|
| 指标 | 使用 Vercel AI SDK v5 的 streamObject，禁止 fetch + 手动解析 SSE |
| 验证方式 | 代码审查确认使用 streamObject API |
| 渲染模式 | 使用 streamObject 的 `partialObject` 增量更新 UI：每到达一个新字段即触发一次渲染，未到达字段保持骨架屏；通过 `onError` 回调捕获流式过程中的错误并展示 |

### NFR-4: Schema 共享

| 字段 | 描述 |
|------|------|
| 指标 | 响应结构 schema 使用 Zod 定义在 shared/schemas/ 目录，前后端共用 |
| 验证方式 | 前后端 import 自同一 schema 文件 |

### NFR-5: 样式

| 字段 | 描述 |
|------|------|
| 指标 | 所有颜色/间距使用 @theme tokens，禁止硬编码 |
| 验证方式 | 代码中无硬编码 color/hex 值 |

### NFR-6: 响应式

| 字段 | 描述 |
|------|------|
| 指标 | 最小支持 768px，核心体验在 1280px+ |
| 验证方式 | 浏览器窗口缩放测试 |

### NFR-7: 卡片间距

| 字段 | 描述 |
|------|------|
| 指标 | 卡片之间间距 24px（遵守视觉宪法第 #15 条） |
| 验证方式 | 视觉检查 |

## 6. 关键实体

### AIResponse (结构化响应)

```
AIResponse
  ├── thinkingProcess: ThinkingStep[]    // 思考过程，流式到达
  ├── toolCalls: ToolCall[]              // 工具调用列表，流式到达
  ├── finalAnswer: string                // 最终答案 markdown，流式到达
  └── metadata: ResponseMetadata         // 响应元信息
```

### ResponseMetadata (响应元信息)

```
ResponseMetadata
  ├── modelUsed: string        // 实际使用的模型 ID
  ├── totalTokens: number      // 总 token 消耗（input + output）
  ├── latencyMs: number        // 端到端延迟（ms）
  └── finishedAt: number       // 完成时间戳（Unix ms）
```

### ThinkingStep (思考步骤)

```
ThinkingStep
  ├── id: string
  ├── content: string
  └── timestamp: number
```

### ToolCall (工具调用)

```
ToolCall
  ├── id: string
  ├── name: string
  ├── input: Record<string, unknown>
  ├── output: string
  └── duration: number
```

### Model (模型选项)

```
Model
  ├── id: string           // 唯一标识
  ├── name: string         // 展示名称
  ├── provider: string     // 提供商
  └── icon?: ReactNode     // 可选图标
```

## 7. 假设

- 使用的 AI API provider 均支持结构化输出（structured output）格式
- Edge Runtime 环境下可以访问所选 provider 的 API 端点
- Vercel AI SDK v5 与项目已安装的 React 19 + Next.js 15 兼容
- 用户无需登录即可使用 Playground（无鉴权拦截）
- **API Key 存储在服务器端环境变量中**，前端不接触 Key，通过 Next.js API Route 代理调用
- 字符上限统一按 8000 字符估算（约 2000-4000 tokens，因模型而异），足够覆盖常见 prompt 场景
